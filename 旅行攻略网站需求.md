# 南京旅行攻略网站 — 需求文档

## 1. 项目概述

一个面向移动端的轻量级南京旅行攻略网站，以景点浏览和旅行信息查阅为核心功能。采用纯前端方案，数据通过 Vue 3 + Mock 模拟实现，无需后端服务。

| 项目 | 内容 |
|---|---|
| 项目名称 | 南京旅行攻略 |
| 目标平台 | 移动端（手机 / 平板） |
| 技术栈 | Vue 3 + Vite + Vue Router + Pinia + Mock.js |
| 数据方案 | 前端 mock 数据（本地 JSON + Mock.js） |
| 部署方式 | 静态站点，可部署至 GitHub Pages / Vercel / Netlify |

## 2. 目标用户

- 计划前往南京旅游的自由行游客
- 在南京本地想了解周边景点信息的用户
- 对南京文化历史感兴趣的浏览者

## 3. 页面结构

### 3.1 首页（景点推荐）

- 顶部轮播 Banner（3–4 张南京标志性景点图片）
- "热门景点" 横向滑动卡片区（6–8 个景点）
- "按区域浏览" 快捷入口（秦淮区、玄武区、鼓楼区、栖霞区）
- "今日推荐" 区域：随机展示 1 个精选景点

### 3.2 景点列表页

- 列表形式展示所有景点（图片 + 名称 + 评分 + 简介）
- 顶部搜索框（按景点名称 / 关键词搜索）
- 筛选栏：按区域、按类别（自然风光 / 历史人文 / 宗教寺庙 / 博物馆）
- 排序：按热门度 / 评分 / 距离
- 支持下拉加载更多（分页模式）

### 3.3 景点详情页

- 景点大图 + 基本信息（名称、评分、开放时间、门票价格、建议游玩时长）
- 景点简介 / 历史背景
- 交通指南（地铁 + 公交）
- 周边推荐（附近的景点 / 美食）
- "收藏" 按钮

### 3.4 旅行贴士页

- 最佳旅行季节
- 推荐游玩路线（1 日游 / 2 日游 / 3 日游）
- 本地美食推荐
- 住宿建议（区域推荐）

### 3.5 我的收藏页

- 收藏的景点列表（从本地 localStorage 读取）
- 点击跳转至详情页
- 左滑取消收藏

### 3.6 App 布局

- 底部 Tab 导航：首页 | 景点 | 贴士 | 收藏
- 顶部状态栏显示当前页面标题

## 6. 非功能需求

- **移动优先**：设计以 375px–414px 宽度为基准，适配主流手机屏幕
- **响应式**：平板端（768px+）适配双栏布局
- **性能**：首屏加载时间 < 2s（静态站点）
- **离线能力**：收藏数据写入 localStorage，页面基础内容可离线查看
- **SEO**：使用 vue-router Hash 模式，配合预渲染或 SSG 优化
